# Facturas Inteligentes — ML Pipeline

Invoice classification ML pipeline: data quality, preprocessing, and supervised training
with TensorFlow. Runs locally via `python -m` or inside Docker for a reproducible
environment.

## Quick Path

```bash
# Local
pip install -r requirements.txt
pytest tests/ -v
# Recommended (new package structure):
python -m src.data.pipeline
python -m src.preprocessing.pipeline
python -m src.training.pipeline

# Docker
docker build -t facturas .
docker run --rm -v .\data:/app/data facturas pytest tests/ -v
# Recommended (new package structure):
docker compose run --rm pipeline python -m src.data.pipeline
docker compose run --rm pipeline python -m src.preprocessing.pipeline
docker compose run --rm pipeline python -m src.training.pipeline
```

## Local Execution

### Run Tests

```bash
pytest tests/ -v
```

All 125 tests must pass before committing changes.

### Run Pipelines

```bash
# Data quality report → results/quality_report.json
python -m src.data.pipeline

# Preprocessing → models/preprocessor_supervisado.pkl + results/preprocessing_report.json
python -m src.preprocessing.pipeline

# Training → models/modelo_supervisado.keras + results/training_report.json
python -m src.training.pipeline
```

## Docker Setup

```bash
# Build the image (first time: ~2-5 min with TensorFlow)
docker build -t facturas .
```

## Docker Commands

### Run Tests in Docker

```bash
# Full suite (data volume required for dataset-dependent tests)
docker run --rm -v .\data:/app/data facturas pytest tests/ -v

# Or via compose
docker compose run --rm pipeline pytest tests/ -v
```

### Run Pipelines in Docker

| Pipeline | Command | Output |
|----------|---------|--------|
| Quality report | `docker compose run --rm pipeline python -m src.data.pipeline` | `results/quality_report.json` |
| Preprocessing | `docker compose run --rm pipeline python -m src.preprocessing.pipeline` | `models/preprocessor_supervisado.pkl`, `results/preprocessing_report.json` |
| Training | `docker compose run --rm pipeline python -m src.training.pipeline` | `models/modelo_supervisado.keras`, `results/training_report.json` |

All pipeline outputs are written to host directories via bind mounts and persist across
container runs.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| `ModuleNotFoundError: No module named 'src'` | `PYTHONPATH` not set or running from wrong directory | Run from project root; Docker sets `PYTHONPATH=/app` automatically |
| `docker: permission denied` | Docker Desktop not running or user not in `docker-users` group | Start Docker Desktop; add user to group and log out/in |
| `FileNotFoundError: data/raw/...` | Bind mount missing or wrong path | Use absolute paths or verify `./data` exists on host |
| Tests fail with `tensorflow` import errors | Missing system libraries in slim image | Rebuild image; slim-bookworm includes required `glibc` |
| `docker compose` unknown command | Compose v1 installed instead of plugin | Use `docker-compose` (hyphen) or upgrade to Docker Compose v2 |
